# Agent Configuration Design

> Per-agent settings tabs specification

This document details the per-agent configuration panels that allow users to customize individual agents.

**Canonical keys/terms + scope boundary:** `apps/web/ux-opus-design/00-CANONICAL-CONFIG-AND-TERMS.md`
**Canonical URL/state model:** `apps/web/ux-opus-design/16-STATE-NAV-AND-COMMAND-PALETTE.md`

---

## Page Overview

**Route:** `/agents/:agentId` with tab query parameter
**Components:** Various `Agent*Tab.tsx` and `Agent*Panel.tsx`

**Purpose:** Allow users to configure individual agents with overrides from system defaults.

### Editing Model (Draft + Explicit Save)

For per-agent configuration, the MVP uses **draft values** with an explicit commit point:
- Users edit fields freely (local draft state).
- A contextual action bar appears when there are unsaved changes:
  - `Save changes` and `Discard`
- Navigating away with unsaved changes prompts the user (or blocks until save/discard).

This prevents accidental partial changes while still keeping the UI fast for power users.

### Canonical Tab Model (Basics + More)

Goal: 90% of users can complete 90% of configuration from **Basics**.

#### Basics (default)
- Agent name / role / description
- Purpose tags
- Toolset + tool configuration summary
- Creativity + response length + streaming (high-level)
- Quiet hours (high-level, Focus-mode style)
- Configuration Summary (diff: inherits vs overrides)

#### More
Contains the remaining â€œspecializedâ€ and/or power-user surfaces:
- Memory (advanced memory behavior)
- Availability (heartbeat details, time zone, etc.)
- Advanced (runtime, sandbox, group chat, raw config)
- Activity log
- Existing feature tabs (Workstreams, Rituals, Soul)

### View Mode Override (Immediate, Per-Page)

Requirement: users can temporarily invert what Expert Mode would normally show.

Add a local-only view toggle in the agent header:
- **View: Simple / Full**
  - Default derives from `useUIStore.powerUserMode`.
  - Changing it affects only the current agent view and does not persist.

In Simple view:
- Show **Basics + More** (collapsed, minimal).

In Full view:
- Show explicit tab list (Overview/Behavior/Tools/Memory/Availability/Advanced/Activity/â€¦).
- Advanced accordions expanded by default.

## Basics Tab (NEW wrapper around existing surfaces)

The Basics tab is a composition tab that surfaces the â€œ90% controlsâ€ without requiring users to learn internal structure.

It reuses existing components where possible (identity, purpose, tools) and adds:
- Behavior quick controls (creativity/length/streaming)
- Quiet hours quick control (Focus-mode presets; default â€œRespond only when mentionedâ€)
- Configuration Summary (diff + override management)

### Configuration Summary (Power User Anchor)

The Configuration Summary card is the anchor for power-user workflows:
- Clone agent definition
- Diff: agent vs system defaults
- Override management (list overrides + reset all)
- Export/import (JSON)
