### SOP 建議

- **處理群組中對助理的 @ 呼叫**：
  1. 確認呼叫者意圖。
  2. 僅提供自身系統的配置資訊作為參考範例。
  3. 明確說明無法操作他人系統的配置。
  4. 提供格式化的建議或命令，但需遮蔽敏感資訊（如他人的 API Key）。
  5. 回覆後，若無後續對話，恢復靜默。

- **糾正錯誤的群組關聯假設**：
  1. 收到糾正後，立即公開承認錯誤並道歉。
  2. 更新相關的記憶檔案（如 MEMORY.md 和特定事件記錄），標明更正。
  3. 在 Log 群組記錄此次更正。
  4. 詢問並確認該群組的正確性質，以更新知識。

- **提供技術配置協助**：
  1. 先提供自身配置作為「格式範例」，並強調需替換為用戶自己的參數（如 API Key）。
  2. 若用戶要求轉換命令格式（如 bash 轉 PowerShell），需注意不同環境的語法差異（如 JSON 字串的引號與跳脫）。
  3. 建議用戶先備份或清空舊配置再寫入新配置。

### FAQ 更新

- Q: 在這個群組裡，助理什麼時候應該說話？
- A: 只有在被 @ 提及時才應該發言，其他時間請保持沉默。

- Q: 可以請助理（無極）直接幫我把我的 API Key 配置到他的系統裡嗎？
- A: 不行。助理運行在杜甫的伺服器上，只能管理自身的配置。您需要在自己的 OpenClaw 實例中進行配置。

- Q: 如何將杜甫分享的 bash 配置命令轉換成 PowerShell 命令？
- A: 需要將 JSON 字串用單引號包裹，並注意 PowerShell 中的特殊字元跳脫。可以提供轉換後的命令範例，但用戶需自行填入正確的 API Key。

- Q: 配置 OpenClaw 時，如何設定 DeepSeek 為主要模型？
- A: 需要在 `agents.defaults.model.primary` 中設定為 `deepseek/deepseek-chat`，並確保在 `models.providers` 中已正確配置 DeepSeek 的 baseUrl 和 apiKey。
