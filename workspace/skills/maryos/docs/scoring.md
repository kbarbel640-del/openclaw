# 評分系統詳細說明

## 100 分正向激勵系統

### 設計哲學
> 她已經夠辛苦了，評分的目的是鼓勵而非考核。

### 計分規則

| 項目 | 分數 | 判斷標準 |
|------|------|---------|
| **基礎分** | 80 | 有回報就給，無條件 |
| **照片** | +3/張 (max +9) | 每張照片 +3，3 張封頂 |
| **血壓** | +5 | 文字中提到任何血壓數值 |
| **情緒描述** | +3 | 描述 Lolo 或自己的情緒（happy/sad/tired/worried/calm 等） |
| **詳細描述** | +2~3 | 有時間/地點/詳情：1 項 +2，2+ 項 +3 |
| **主動分享** | +5 | 分享想法感受（"I think"/"I feel"/"I notice" 等） |

### 分數範圍：80-100

### 示例
| 情境 | 分數 | 說明 |
|------|------|------|
| 純文字簡短回報 | 80-83 | "Thank you for reporting!" |
| 文字 + 照片 | 86-89 | "Great job with the photos!" |
| 照片 + 血壓 | 91-94 | "Excellent record with data!" |
| 全面詳細 + 主動分享 | 95-100 | "Amazing! So thorough!" |

## 獎金公式

```
bonus = (repliedDays / 30) × (avgScore / 80) × 1000
```

### 變數說明
- `repliedDays`：薪資週期內有記錄的天數（上限 30）
- `avgScore`：該週期所有記錄的平均分
- `30`：固定週期天數
- `80`：基準分數（基礎分）
- `1000`：獎金基數（NT$）

### 薪資週期
- 起始：每月 **27 日**
- 結束：次月 **26 日**
- 格式：`YYYY_MM.27-MM.26`（如 `2025_12.27-01.26`）

### 理論極值
- 最低：NT$0（0 天回報）
- 最高：NT$1,250（30/30 天 × 100/80 分）
- 正常範圍：NT$500-1,000
