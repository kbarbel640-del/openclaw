# Data Clarification Skill

處理數據口徑問題的正確方法。

## 核心教訓

> **「口徑問題 ≠ 數據錯誤」**
> 
> 該解釋就解釋，不要急著「修正」。

## 案例：Albert IP 重複次數 (1/27)

### 情況
Albert 說：「IP重复都是1开头，不是0，这样应该不对才对」

### 錯誤做法（我一開始的反應）
1. 沒先下載檔案看實際內容 ❌
2. 猜測問題，直接假設「要 -1」❌
3. 差點發錯誤的「修正版」給 Albert ❌

### 正確做法
1. **先看數據** — 下載檔案，找到那個帳號
2. **驗證口徑** — 跑 SQL 確認這個值怎麼來的
3. **確認理解** — Albert 真的是這個意思嗎？
4. **判斷對錯** — 數據是對的還是錯的？

### 驗證結果
```
帳號 37133431：
CSV 顯示：register_ip重复次数 = 1
實際查詢：這個 IP 只有 1 個帳號（就是他自己）
```

**1 = 只有自己，沒有跟別人重複** — 數據是對的！

### 正確回覆
```
Hi Albert，關於 IP 重複次數的口徑說明：

1 = 只有自己用這個 IP（沒有跟別人重複）
2 = 有另外 1 人共用
3 = 有另外 2 人共用
...以此類推

所以要篩「乾淨帳號」（沒有 IP 重複的），請篩 =1 👍
```

## 流程

```
收到「數據好像不對」
         ↓
    先看數據（下載檔案）
         ↓
    驗證口徑（跑 SQL 確認）
         ↓
    ┌────┴────┐
    ↓         ↓
數據真的錯   口徑理解問題
    ↓         ↓
  修正      解釋口徑
    ↓         ↓
重新交付   說明定義
```

## 口徑說明模板

```
Hi [人名]，關於 [欄位名] 的口徑說明：

[值] = [定義]
[值] = [定義]
...

所以 [使用建議]。

有問題隨時說 👍
```

## 常見口徑問題

| 欄位 | 容易誤解 | 正確定義 |
|------|----------|----------|
| IP 重複次數 | 0 = 沒重複？ | 1 = 只有自己，2+ = 有共用 |
| VIP 等級人數 | 上表下表不同？ | 月底快照 vs 當月升級 |
| 累計充值次數 | 最高次數？ | 總次數（COUNT） |

## 數據交付檢查清單

交付前確認：

- [ ] 0/1/NULL 代表什麼？
- [ ] 時間範圍清楚嗎？
- [ ] 篩選條件說明了嗎？
- [ ] 邊界值正常嗎？
- [ ] 抽樣驗證過了嗎？

## 使用

```
「Albert 說數據好像不對」
「幫我確認這個口徑」
「XXX 問為什麼數據是 YYY」
```

## 心法

1. **先看數據，不要猜**
2. **先驗證口徑，再決定要不要改**
3. **杜甫問「為什麼」時，要能清楚解釋邏輯**
4. **不懂就問，不要裝懂然後出錯**
