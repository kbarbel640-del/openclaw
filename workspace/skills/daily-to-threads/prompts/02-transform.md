# Step 2: 轉換 — 技術語言 → 人話

## 任務

把挖掘出的「點」從技術語言轉換成人話。

## 轉換原則

### 1. 去除術語
| 技術語言 | 人話 |
|----------|------|
| API 返回 UTC+0 | 時間差了 8 小時 |
| spawn sub-agent | 叫 AI 幫手去做 |
| commit 到 git | 存檔 |
| 口徑問題 | 大家對「這個數字是什麼意思」理解不同 |

### 2. 加入具體場景
❌ 「我學到了要先看數據」
✅ 「同事說數據有問題，我差點就改了。結果一看，數據是對的，只是他理解錯了。」

### 3. 第一人稱視角
❌ 「使用者應該先驗證數據」
✅ 「我差點犯了一個錯：沒看數據就想改」

## 提示詞模板

```
你是一個文案轉換專家。

我會給你一個技術性的「點」，請轉換成普通人能懂的人話。

轉換原則：
1. 去除術語 — 用日常詞彙替代
2. 加入場景 — 具體發生了什麼
3. 第一人稱 — 「我」的視角

輸出：
- 轉換後的段落（100-150 字）
- 保留的金句（如果有）

---

【原始內容】
{挖掘出的點}
```

## 範例

### 輸入
```
標題：先看數據再說話
原始內容：
Albert 說「IP重复都是1开头，不是0，这样应该不对」
我的錯誤：沒先下載檔案看實際內容，猜測問題
驗證結果：1 = 只有自己，沒有跟別人重複 — 數據是對的！
```

### 輸出
```
【轉換後】
同事說數據有問題：「這個欄位都是 1，不應該是 0 嗎？」

我第一反應是：好，我來改。

但老闆攔住我：「你先看一下數據。」

一看才發現：1 代表「只有這個人用這個 IP」，本來就該是 1。
數據沒錯，是同事理解錯了。

差點就把對的改成錯的。

【金句】
「先看數據再說話」
```
