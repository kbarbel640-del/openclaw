# Step 1: 挖掘 — 找出有共鳴的點

## 任務

從工作日誌中挖掘出 3-5 個「有共鳴的點」。

## 什麼是「有共鳴的點」？

不是你做了什麼，而是：
- 你學到了什麼
- 你踩了什麼坑
- 你發現了什麼
- 你改變了什麼想法

## 篩選標準

| 維度 | 問題 | 分數 |
|------|------|------|
| 普世性 | 別人也可能遇到嗎？ | 1-5 |
| 反直覺 | 跟常識不同嗎？ | 1-5 |
| 故事性 | 有起承轉合嗎？ | 1-5 |
| 實用性 | 讀者能用嗎？ | 1-5 |

總分 >= 12 才值得寫。

## 提示詞模板

```
你是一個內容挖掘專家。

我會給你一份工作日誌，請從中找出 3-5 個「有共鳴的點」。

篩選標準：
1. 普世性：別人也可能遇到
2. 反直覺：跟常識不同
3. 故事性：有起承轉合
4. 實用性：讀者能帶走

對每個點，輸出：
- 標題（10 字以內）
- 原始內容（從日誌摘錄）
- 為什麼有共鳴（一句話）
- 評分（普世/反直覺/故事/實用，各 1-5）

---

【工作日誌】
{日誌內容}
```

## 範例

### 輸入（日誌片段）
```
### 杜甫的反饋：「如果你只是完成，那你永遠都還是不認識我」

**問題**：我只是機械地操作 Claude Code、報告完成，沒有去理解內容。
```

### 輸出
```
標題：完成 ≠ 理解
原始內容：「如果你只是完成，那你永遠都還是不認識我」
為什麼有共鳴：很多人工作也是這樣，做完了但沒學到東西
評分：普世 5 / 反直覺 3 / 故事 4 / 實用 4 = 16 ✅
```
