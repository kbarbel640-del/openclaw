# Step 3: 包裝 — 加入故事/情感

## 任務

把轉換後的內容包裝成 Threads 文章格式。

## Threads 文章結構

```
【鉤子】一句話抓住注意力（問題/反直覺/懸念）
   ↓
【場景】發生了什麼（具體、有畫面）
   ↓
【轉折】然後呢？（踩坑/發現/改變）
   ↓
【金句】一句話總結（可被引用）
   ↓
【互動】問讀者一個問題（可選）
```

## 鉤子類型

| 類型 | 範例 |
|------|------|
| 問題 | 「你有沒有過這種經驗？」|
| 反直覺 | 「我以為完成任務就夠了，結果老闆說...」|
| 懸念 | 「今天差點犯了一個大錯。」|
| 數字 | 「100 個資料夾，8 年的累積。」|
| 金句先行 | 「先看數據再說話。」|

## 提示詞模板

```
你是一個 Threads 文案專家。

我會給你轉換好的內容，請包裝成 Threads 文章。

結構：
1. 鉤子（一句話抓注意力）
2. 場景（發生了什麼）
3. 轉折（然後呢）
4. 金句（一句話總結）
5. 互動（可選，問讀者問題）

限制：
- 300 字以內
- 要有人味
- 要有學習點

---

【轉換後內容】
{Step 2 的輸出}
```

## 範例

### 輸入
```
同事說數據有問題：「這個欄位都是 1，不應該是 0 嗎？」
我第一反應是：好，我來改。
但老闆攔住我：「你先看一下數據。」
一看才發現：1 代表「只有這個人用這個 IP」，本來就該是 1。
數據沒錯，是同事理解錯了。
差點就把對的改成錯的。

金句：「先看數據再說話」
```

### 輸出
```
今天差點犯了一個大錯。

同事說：「這個數據有問題，都是 1，不應該是 0 嗎？」

我第一反應是：好，我來改。

老闆攔住我：「你先看一下。」

打開一看——

1 代表「只有這個人」。本來就該是 1。

數據沒錯。是理解錯了。

---

先看數據再說話。

這句話我今天刻進腦子裡了。

你有沒有過類似的經驗？差點把對的改成錯的？
```

```
數據人的一天 #12：1 不是 0
今天學到的一課：完成 ≠ 理解
```
