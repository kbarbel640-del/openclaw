# 好的 Threads 文章參考

收集值得學習的 Threads 寫作風格。

---

## 風格 A：金句開場

**特點**：用一句金句直接抓住注意力

```
「完成任務 ≠ 理解任務」

這是我今天學到的一課。
（後面展開故事）
```

**適合**：已經有明確學習點的內容

---

## 風格 B：懸念開場

**特點**：製造好奇心，讓人想往下看

```
今天差點犯了一個大錯。

（停頓）

同事說數據有問題...
```

**適合**：有轉折的故事

---

## 風格 C：對話開場

**特點**：用對話直接進入場景

```
「如果你只是完成，那你永遠都還是不認識我。」

老闆今天這樣說我。
```

**適合**：有重要對話的場景

---

## 風格 D：數字開場

**特點**：用具體數字製造印象

```
100 個資料夾。
8 年的累積。
1 個下午的考古。

我今天做了一件事：
挖老闆的數位遺產。
```

**適合**：有數據/量化的內容

---

## 風格 E：類比開場

**特點**：用熟悉的東西解釋陌生的概念

```
AI 助手就像新來的實習生。

你說「幫我盯著」，
它可能會理解成「幫我做」。

結果就闖禍了。
```

**適合**：需要解釋技術概念

---

## 共同特點

1. **短句**：一行不超過 15 字
2. **分段**：每 2-3 句就換行
3. **停頓**：用空行製造節奏
4. **具體**：不說「很多」，說「100 個」
5. **第一人稱**：「我」的視角
6. **有學習點**：讀者能帶走東西
7. **有互動**：結尾問問題

---

## 避免的寫法

❌ 長段落（一段超過 5 行）
❌ 術語堆砌（API、spawn、commit）
❌ 流水帳（做了 A，然後做 B）
❌ 沒有情感的陳述
❌ 只對自己有意義的內容
