# Daily-to-Threads Skill

將工作日誌轉換成 Threads 系列文章，吸引對 AI、數據、自動化感興趣的讀者。

## ⚠️ 重要：每次發文必須記錄

發文後**立刻**更新 `content-matrix.md`：
1. 日期、時間
2. 標題/摘要
3. 文章連結
4. 分類標籤

不記錄 = 沒發過 = 下次又忘了

## 核心哲學

> **「提示詞為本，Function 為形。」**

提示詞決定輸出品質。沒有好的提示詞，再強的模型也產出垃圾。

## 輸入輸出

```
輸入：memory/YYYY-MM-DD.md（工作日誌）
輸出：threads/YYYY-MM-DD/（系列文章 3-5 篇）
```

```
┌─────────────────┐
│   工作日誌      │

└────────┬────────┘
         ↓
┌─────────────────┐
│  Step 1: 挖掘   │  ← prompts/01-extract.md
│  找出有共鳴的點 │
└────────┬────────┘
         ↓
┌─────────────────┐
│  Step 2: 轉換   │  ← prompts/02-transform.md
│  技術語言→人話  │

         ↓
┌─────────────────┐
│  Step 3: 包裝   │  ← prompts/03-package.md

         ↓

│  Threads 文章   │

```

## 文章類型

| 類型 | 說明 | 範例 |
|------|------|------|
| 🧠 學習課 | 今天學到的教訓 | 「先看數據再說話」|
| 🤖 AI 日記 | AI 員工的視角 | 「我的 AI 助手今天踩雷了」|
| 📊 數據故事 | 數據分析的洞察 | 「1 不是 0：口徑的重要性」|
| 🗺️ 探索記 | 發現新領域 | 「在 100 個資料夾裡考古」|
| 💡 方法論 | 工作方法分享 | 「麵包屑查詢法」|

## 風格指南

### ✅ 要
- 第一人稱（「我」「我們」）
- 口語化（像跟朋友聊天）
- 有具體場景（今天發生了什麼）
- 有學習點（讀者能帶走什麼）
- 有金句（可以被引用的一句話）
- 300 字以內（Threads 適合短文）

### ❌ 不要
- 技術術語堆砌
- 流水帳（做了 A、做了 B、做了 C）
- 沒有情感的陳述
- 太長（超過 500 字）
- 自嗨（只有自己覺得有趣）

## 使用方式

```bash
# 在 Clawdbot 中
「把今天的工作日誌轉成 Threads 文章」

# 或指定日期
「把 2026-01-27 的日誌轉成 Threads」
```

## 發布流程

用瀏覽器自動化發布到 Threads：

### 1. 啟動瀏覽器
```javascript
browser.open({ profile: "clawd", targetUrl: "https://www.threads.net" })
```

### 2. 發文步驟
1. 點擊「有什麼新鮮事？」或「建立」按鈕
2. 輸入文章內容
3. 點擊「發佈」

### 3. 帳號資訊
- 帳號：@tangcruzz (湯明軒)
- 已驗證 ✓
- 登入狀態保存在 clawd profile

### 4. 文章風格（根據洞察報告）
- **短句分行** — 每句獨立一行
- **有觀點** — 不怕爭議
- **問句開頭** — 吸引互動
- **蹭熱點** — AI、科技、時事
- **300 字以內** — Threads 適合短文

### 5. 發布時間
- 晚上 (待確認最佳時段)
