## 三、各工作范围与指标的关系

> 不是「谁负责什么指标」，而是「你的工作成果通常会在哪些指标上体现」。大家一起推，一起看。

### 🔵 市场推广
工作成果主要体现在：**注册人数、首充人数、首充转化率**
- 投放渠道越精准 → 注册质量越高 → 首充转化越好
- 活动策划的吸引力直接影响新用户愿不愿意掏钱

### 🟢 支付优化
工作成果主要体现在：**充值总额、渠道成功率、提现处理效率**
- 渠道排序合理 → 成功率↑ → 充值↑
- 出款快 → 用户信任↑ → 留存↑

### 🟣 VIP / CRM
工作成果主要体现在：**人均充值、复充率、VIP 贡献占比**
- VIP 升级策略做得好 → 高价值玩家留住 → 人均充值↑
- 召回活动精准 → 流失用户回来 → 活跃↑

### 🟡 产品
工作成果主要体现在：**活跃人数、投注总额、用户参与率**
- UI/文案优化 → 用户体验好 → 留下来的人多
- 新游戏品质好 → 投注多样性↑ → 总投注↑

### ⚪ 运营统筹
关注全局：**所有指标的趋势和异常**
- 资源怎么分配、优先级怎么排、团队怎么协调
- 周进度对齐，及时发现偏差

## 四、数据口径（查数据时参考）

| 指标 | 数据表 | 字段 | 注意事项 |
|------|--------|------|---------|
| 注册 | channel_data_statistics | register_number | 当日新注册 |
| 活跃 | channel_data_statistics | active_number | 当日登录独立数 |
| IP | channel_data_statistics | ip_number | ⚠️ 不能从 logininfor 推算 |
| 充值 | channel_data_statistics | recharge_amount | order_status=1 |
| 提现 | channel_data_statistics | withdraw_amount | ⚠️ order_status=3，用 finish_time |
| 总投注 | channel_game_statistics | bet_amount | ⚠️ 不用 player_statistics_day |
| 首充 | channel_data_statistics | first_recharge_number | ⚠️ 不用 first_deposit_record |
| 充值人数 | channel_data_statistics | recharge_number | 当日有充值的独立数 |
