## 1. 執行摘要

### 一句話定位

**「一個人 + 五個AI，公開建一間公司。全程直播。」**

### 使命

用真實的創業過程證明：一個有經驗的人搭配 AI 團隊，能做到過去需要 10-20 人團隊才能做的事——然後把方法教給更多人。

### 願景

三年內成為台灣「AI 實戰應用」的第一品牌。不靠理論，靠每天公開的真實營運數據說話。

### 核心差異化

| 別人怎麼做 | ThinkerCafe 怎麼做 |
|---|---|
| 教 AI 工具操作 | 公開展示 AI 如何跑一間公司 |
| 講師背景是「研究 AI」 | 創辦人背景是「管過 300 人團隊」+ 經濟部 iPAS 11 年專家委員 |
| 課程賣完就結束 | 學員直接進入運轉中的 AI 系統觀摩 |
| 內容靠想像 | 內容 100% 來自每日真實工作記錄 |

### 商業模式一句話

**工作產生內容 → 內容吸引觀眾 → 觀眾變學員 → 學員產生案例 → 案例變新內容。Cruz 負責決策和教學，AI 負責其他所有事。**

## 6. 風險與對策

### 風險 1：API 成本失控

| | 說明 |

| **發生機率** | 中（爆量使用或模型漲價） |
| **影響** | 月成本從 15K 暴增到 50K+ |

| **應急方案** | 立即降級非必要 AI 功能；把用量排名前 3 的任務做成本最佳化 |

### 風險 2：內容產出斷裂

| | 說明 |

| **預防機制** | ① 維持 2 週的內容庫存（Buffer）② 設計不依賴 Cruz 的內容類型（AI 自動生成的市場觀察、工具評測）③ 「人機對話實錄」可批次產出 |

### 風險 3：學員轉化率低

| | 說明 |

### 風險 4：Cruz 精力不足

| **應急方案** | ① 暫停新課開發，只維護現有內容 ② 把非核心工作外包（設計、剪輯）③ 90 天後評估是否招一位兼職人類助理 |

## 附錄：關鍵成功因素

1. **Cruz 的紀律**：每天 5 分鐘口述 > 偶爾寫 2000 字長文。穩定輸入才有穩定輸出。
2. **AI 的邊界意識**：知道什麼該自動做、什麼必須等 Cruz 決策。不擅作主張。
3. **公開透明的勇氣**：這是最大的差異化。數字好看就秀，數字難看也秀。信任來自真實。
4. **飛輪優先於功能**：不追求每個部門都完美，追求閉環在轉。轉起來再優化。
5. **成本極度控制**：AI 工具月成本控制在 20K 以內。賺錢之前先確保不燒錢。
