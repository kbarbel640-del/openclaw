# 快速輸錢玩家 PR 分群分析
## 2026 年 1 月數據

### 📌 分析目的
找出「輸錢速度快」的玩家群體，用 PR 百分位排名分群，評估是否值得做精準召回。

### 📐 計算口徑

**數據來源**：`player_statistics_day`（每日玩家統計）+ `sys_player`（玩家主表）

**篩選條件**：
- 時間範圍：2026-01-01 ~ 2026-01-31
- 1 月有充值記錄（`recharge_amount > 0`）
- 1 月整月為淨虧損（`SUM(profit_amount) < 0`）
- 共篩出 **12,242 人**

**核心指標 — burn_per_day（每日燒錢速度）**：
```
burn_per_day = |月總虧損| ÷ 活躍天數
```
- `月總虧損` = SUM(profit_amount)，取絕對值
- `活躍天數` = COUNT(DISTINCT statistics_day)，即該玩家在 1 月有數據的天數（不是日曆天）
- **含義**：平均每個活躍日虧掉多少錢（₹）

**輔助指標 — loss_pct（虧損佔充值比）**：
```
loss_pct = |月總虧損| ÷ 月總充值 × 100%
```
- 超過 100% 代表虧的比充的多（可能有上月餘額）

### 📊 PR 分群定義

按 `burn_per_day` 從低到高排序，用百分位數切割：

### ⚠️ 已知限制

1. **日級精度**：只能算到「幾天輸光」，無法算「幾分鐘」（DB 無餘額歷史快照）
2. **活躍天數 ≠ 日曆天數**：如果玩家 1/1 和 1/31 各玩一天，active_days=2，但實際跨了 30 天
3. **不區分遊戲類型**：百家樂和體育的輸錢速度天然不同，此版未拆分
4. **上月餘額效應**：如果玩家帶著 12 月的餘額進 1 月，loss_pct 可能 > 100%
5. **只看虧損玩家**：盈利玩家（贏家）未納入本次分析

| lifetime_recharge | 歷史總充值 |

| pr_group | PR 分群標籤 |
