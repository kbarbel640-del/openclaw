# 提示詞庫 V1.0 規格文件

## 版本資訊
- 版本號：1.0
- 建立日期：2025-01-19
- 目標：建立公司第一版核心提示詞庫，涵蓋基礎組件、核心配方與常用範本

## 核心架構
```
prompt-library/
├── components/     # 組件庫：最小可複用單位
├── recipes/        # 配方庫：組件組合的任務流程
├── templates/      # 範本庫：預填好的高頻任務模板
└── spec/          # 規格文件
```

## 一、組件庫 (components/) - 10個核心組件

### 角色畫像類
1. **persona-founder.v1.md** ✓
   - 描述：創辦人畫像，體現系統架構師型領導者

2. **persona-pm.v1.md** ✓

3. **persona-dev-lead.v1.md** ✓

### 語氣風格類
4. **tone-ambitious-but-grounded.v1.md** ✓

   - 相依性：無

### 輸出格式類
5. **format-markdown-report.v1.md** ✓
   - 描述：Markdown報告格式，結構清晰、層次分明

6. **format-user-story-g-w-t.v1.md**

7. **format-api-spec-rest.v1.md**

### 思考規則類

   - 描述：Node.js程式碼品質規則，含風格、測試、錯誤處理

    - 描述：逐步思考規則，處理複雜任務前先分析規劃
    - 相依性：無

## 二、配方庫 (recipes/) - 6個核心配方

### 策略層級（L1）
1. **L1-generate-quarterly-strategy.v1.yml**
   - 組件組合：persona-founder.v1, tone-ambitious-but-grounded.v1, format-markdown-report.v1, rules-think-step-by-step.v1
   - 用途：生成季度策略文件

### 產品層級（L2）
2. **L2-create-user-stories-from-epic.v1.yml**
   - 組件組合：persona-pm.v1, format-user-story-g-w-t.v1, rules-invest-principles.v1, rules-think-step-by-step.v1
   - 用途：從史詩任務拆解使用者故事

### 技術設計層級（L3）
3. **L3-design-backend-from-story.v1.yml**
   - 組件組合：persona-dev-lead.v1, format-api-spec-rest.v1, rules-think-step-by-step.v1
   - 用途：從使用者故事設計後端架構

### 實作層級（L4）
4. **L4-implement-api-endpoint.v1.yml**
   - 組件組合：format-api-spec-rest.v1, rules-code-quality-node.v1
   - 用途：實作API端點

5. **L4-debug-ci-cd-failure.v1.yml**
   - 組件組合：rules-think-step-by-step.v1, rules-code-quality-node.v1
   - 用途：除錯CI/CD失敗

### 知識管理（KM）
6. **KM-summarize-meeting-notes.v1.yml**
   - 組件組合：format-markdown-report.v1, tone-concise.v1（待建立）
   - 用途：總結會議記錄

## 三、範本庫 (templates/) - 1個核心範本
