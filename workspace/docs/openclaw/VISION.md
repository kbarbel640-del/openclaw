# OpenClaw — 願景文檔

> 讓每個人都有自己的 AI 員工。

---

## 這是什麼

OpenClaw 不是一個產品，是一個**系統 + 哲學**。

基於 Moltbot（開源 AI agent runtime）+ clawd workspace template，任何人都可以：
1. 在 Telegram/LINE/Discord 上擁有自己的 AI 員工
2. AI 有記憶、會反思、會成長
3. 記憶存在用戶自己的 GitHub repo（隱私由用戶掌控）

## 核心哲學

> Code is disposable, Insight is eternal.
> 提示詞為本，Function 為形。

### SOUL.md — 每個 AI 員工出廠自帶靈魂
- Be genuinely helpful, not performatively helpful
- Have opinions
- Earn trust through competence
- Remember you're a guest

### 信任模型 — 人格擔保，不是技術擔保
- 創辦人（Cruz）的個人資料也跑在同一套系統裡
- 全公司營運全透明
- ThinkerCafe = 認同理念的人聚在一起共享資源的地方

## 漏斗設計

```
LINE/TG 連結 → 體驗 AI 員工 → 進 ThinkerCafe → 聽道理 → 認同 → 留下 → 推薦
```

這不是銷售漏斗，是**信仰漏斗**。留下的人不會走，因為認同的是理念不是功能。

## 增長策略：Influencer Seeding

- 找 10 個 3K-10K 微型 KOL（AI/生產力/自媒體）
- 給免費 seat，不要求發內容
- 產品好 → 他們自然會講
- 零成本，無限槓桿

### Seed Users
1. **Avery** — 接睫毛工作室（預約管理場景）
2. **ThinkerCafe 社群成員** — 已認同價值觀

## 技術棧

| 層 | 工具 | 說明 |
|---|------|------|
| Runtime | Moltbot (開源) | AI agent 執行環境 |
| Workspace | clawd template | SOUL.md + AGENTS.md + skills/ |
| 記憶 | Git + memory/ | 版本化、可回溯、用戶掌控 |
| 通訊 | Telegram/LINE/Discord | 不用裝新 app |
| 備份 | GitHub private repo | 自動 push |

## 記憶系統（核心競爭力）

三流派整合：
1. **Git-Context-Controller** — 版本化記憶，COMMIT/BRANCH/MERGE/CONTEXT
2. **Mem0** — 自動萃取 facts/preferences/relationships，衝突檢測
3. **認知科學** — 遺忘曲線、importance scoring、預測性 context

### 記憶分層（對標 MemoryOS）
| 層 | 檔案 | 功能 |
|---|------|------|
| Core | SOUL.md + USER.md | 不變的身份 |
| Episodic | memory/YYYY-MM-DD.md | 事件記錄 |
| Semantic | MEMORY.md | 長期知識（蒸餾精華）|
| Procedural | skills/ | 技能庫 |
