## 📝 SOP：每日記憶整理

### 1. 閱讀 Telegram 對話

```
時間：每天 09:00, 15:00, 21:00
範圍：指定的工作群組
```

### 2. 識別重要內容

**需要記錄的**：
- ✅ 決策和結論
- ✅ 待辦事項
- ✅ 重要數據
- ✅ 問題和解決方案
- ✅ 學到的經驗

**不需要記錄的**：
- ❌ 閒聊
- ❌ 重複的內容
- ❌ 臨時性的討論

### 3. 整理到 Obsidian

**檔案命名**：`YYYY-MM-DD-主題.md`

**模板**：
```markdown
# [主題]

**日期**：YYYY-MM-DD
**來源**：[群組名稱]
**相關人**：[[人名]]

## 摘要
[用 2-3 句話總結]

## 重點
- 重點 1
- 重點 2

## 決策
- [ ] 待辦事項 1
- [ ] 待辦事項 2

## 連結
- [[相關筆記1]]
- [[相關筆記2]]

## 原文摘錄
> 重要的原始對話內容
```

### 4. 加上標籤和連結

**常用標籤**：
- `#工作/BG666`
- `#工作/報表`
- `#待辦`
- `#決策`
- `#學習`

**建立連結**：
- 提到的人名 → `[[人名]]`
- 提到的項目 → `[[項目名]]`
- 相關筆記 → `[[筆記名]]`

### 5. 提交審核

- 整理完成後，在工作群組回報
- 等待 AI（無極）審核
- 根據反饋修改

## 📝 SOP：監控與回報

### 每小時檢查

1. 打開 666数据日报群
2. 查看最新的小時報
3. 檢查是否有異常（紅色警示、數字異常大/小）
4. 如有異常，立即截圖回報到工作群組

### 異常判斷標準

| 指標 | 正常範圍 | 異常 |
|------|---------|------|
| 優惠比 | 25-35% | <20% 或 >40% |
| 淨流入 | 正數 | 大幅負數 |
| 存款 | >10萬/小時 | <5萬/小時 |

### 回報格式

```
⚠️ 異常回報

時間：HH:MM
指標：[指標名稱]
數值：[異常數值]
正常範圍：[正常範圍]

[截圖]
```

## 📝 SOP：物理操作

### 收到指令時

1. 確認理解指令（不確定就問）
2. 執行操作
3. 截圖回報結果
4. 等待下一步指示

### 回報格式

```
✅ 已完成

操作：[做了什麼]
結果：[截圖]
```

或

```
❌ 無法完成

操作：[嘗試做什麼]
問題：[遇到什麼問題]
[截圖]
```
