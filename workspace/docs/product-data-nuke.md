# 數據核武器 — 產品規格書

> iGaming 數據分析 AI 員工，讓非技術人員也能用自然語言查數據

## 一、產品定位

**一句話**：給 iGaming 公司的「會說話的數據庫」

**目標用戶**：
- 運營主管（Red）：「給我歷史存款 ≥100、筆數 ≥2 的會員名單」
- CRM 專員（Dave）：「首充後 7 天沒二充的人有多少」
- 老闆（Brandon）：「這週存款比上週差多少」

**核心價值**：
- ❌ 不用等數據組排期
- ❌ 不用學 SQL
- ❌ 不用開工單
- ✅ 10 分鐘內拿到數據

---

## 二、核心功能

### 2.1 自然語言查詢
```
用戶：給我 VIP3 以上、最近 30 天沒登入的玩家
AI：查詢中... 找到 2,847 人，要導出 Excel 嗎？
```

### 2.2 自動報表
- 每日存款/提款報表
- 轉化漏斗（註冊→首充→二充→三充）
- 渠道 ROI 分析
- 異常預警（存款驟降、大額提款）

### 2.3 名單導出
- 按條件篩選會員
- 導出 CSV/Excel
- 直接發到 Telegram/企業微信

### 2.4 數據問答
```
用戶：為什麼今天存款比昨天少 20%？
AI：分析中...
    1. VIP3+ 玩家存款下降 35%（主因）
    2. 新註冊數正常
    3. 建議：檢查 VIP3+ 有無大戶流失
```

## 四、部署方式

### 方案 A：SaaS（推薦）
- 客戶提供 DB 只讀帳號
- 我們託管 AI 引擎
- 按月訂閱

### 方案 B：私有部署
- 整套部署到客戶 VPC
- 一次性授權 + 維護費

## 九、風險與對策

| 風險 | 對策 |
|------|------|
| IP 歸屬爭議 | 核心 AI 引擎是 Moltbot，不是工作產出 |
| 數據安全顧慮 | 只讀權限、不存數據、可私有部署 |
| iGaming 法規 | 只做數據分析，不涉及業務操作 |
| 客戶 DB 結構差異大 | 建立 schema mapping 層，標準化 |
