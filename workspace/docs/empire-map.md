## 五行相生循環

```
金 → 水 → 木 → 火 → 土 → 金
```

| 相生 | 含義 |
|------|------|
| 金 → 水 | 玄鑑的品控標準 → 餵給淵識做情報分析 |
| 水 → 木 | 淵識的洞察 → 餵給萌策做內容策略 |
| 木 → 火 | 萌策的內容 → 餵給燎原做社群裂變 |
| 火 → 土 | 燎原的數據 → 餵給厚載做財務結算 |
| 土 → 金 | 厚載的資源 → 餵回玄鑑優化服務品質 |

## 協作機制（方案 C）

**原則：檔案交接 + 一句通知，省 token**

```
玄鑑工作完成
  → 寫入 shared/handoff/金→水/YYYY-MM-DD-task.md
  → Discord 發一句「🌊 淵識，新任務已放置」
  → 淵識讀檔案、處理、寫回結果
  → Discord 回一句「🪞 玄鑑，分析完成」
```

### 協作原則
- 各自獨立作業，有事才 call
- Discord 討論最多 3 回合收斂
- 工作內容走檔案（省 token），Discord 只發通知

### Token 節省估算
- 快速交接（單向通知）：~500 token
- 短討論（3 回合）：~3000 token（Sonnet ≈ $0.01）
- 深度協作：按需，有上限保護

## 部署順序

1. 🪞 玄鑑（已上線）— 幣塔客服品控
2. 🌊 淵識（Q2）— 有品控數據後，開始做競品情報
3. 🌱 萌策（有拉新需求時）
4. 🔥 燎原（有品牌/社群時）
5. ⛰️ 厚載（規模化後）
