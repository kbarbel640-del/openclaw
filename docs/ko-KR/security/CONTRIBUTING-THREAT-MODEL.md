# OpenClaw 위협 모델에 기여

OpenClaw를 더욱 안전하게 만드는 데 도움을 주셔서 감사합니다. 이 위협 모델은 살아있는 문서이며 우리는 누구의 기여도 환영합니다. 보안 전문가가 될 필요는 없습니다.

## 기여 방법

### 위협 추가

우리가 다루지 않은 공격 벡터나 위험을 발견하셨나요? [openclaw/trust](https://github.com/openclaw/trust/issues)에서 이슈를 열고 자신의 말로 설명하세요. 프레임워크를 알 필요도 없고 모든 필드를 채울 필요도 없습니다. 시나리오만 설명하면 됩니다.

**포함하면 유용하지만 필수는 아닙니다.**

- 공격 시나리오와 이를 악용할 수 있는 방법
- OpenClaw의 어떤 부분이 영향을 받는지(CLI, 게이트웨이, 채널, ClawHub, MCP 서버 등)
- 귀하가 생각하는 심각도(낮음 / 중간 / 높음 / 심각)
- 관련 연구, CVE 또는 실제 사례에 대한 링크

검토 중에 ATLAS 매핑, 위협 ID 및 위험 평가를 처리합니다. 이러한 세부 정보를 포함하고 싶다면 좋습니다. 하지만 예상되는 것은 아닙니다.

> **실시간 취약점을 보고하는 것이 아니라 위협 모델에 추가하기 위한 것입니다.** 악용 가능한 취약점을 발견한 경우 책임 있는 공개 지침을 보려면 [신뢰 페이지](https://trust.openclaw.ai)를 참조하세요.

### 완화 제안

기존 위협을 해결하는 방법에 대한 아이디어가 있습니까? 위협을 언급하는 문제나 PR을 엽니다. 유용한 완화는 구체적이고 실행 가능합니다. 예를 들어 "게이트웨이에서 분당 메시지 10개로 발신자당 속도 제한"이 "속도 제한 구현"보다 낫습니다.

### 공격 체인 제안

공격 체인은 여러 위협이 어떻게 현실적인 공격 시나리오로 결합되는지 보여줍니다. 위험한 조합을 발견하면 단계와 공격자가 이를 어떻게 연결하는지 설명하십시오. 실제로 공격이 어떻게 전개되는지에 대한 간략한 설명은 공식적인 템플릿보다 더 가치가 있습니다.

### 기존 콘텐츠 수정 또는 개선

오타, 설명, 오래된 정보, 더 나은 예 - PR을 환영하며 문제가 필요하지 않습니다.

## 우리가 사용하는 것

### 마이터 아틀라스

이 위협 모델은 신속한 주입, 도구 오용, 에이전트 악용과 같은 AI/ML 위협을 위해 특별히 설계된 프레임워크인 [MITRE ATLAS](https://atlas.mitre.org/)(AI 시스템을 위한 적대적 위협 환경)를 기반으로 구축되었습니다. 기여하기 위해 ATLAS를 알 필요는 없습니다. 검토 중에 제출물을 프레임워크에 매핑합니다.

### 위협 ID

각 위협은 `T-EXEC-003`와 같은 ID를 갖습니다. 카테고리는 다음과 같습니다:

| 코드   | 카테고리                  |
| ------ | ------------------------- |
| 정찰   | 정찰 - 정보 수집          |
| 액세스 | 초기 접근 - 진입 획득     |
| EXEC   | 실행 - 악의적인 작업 실행 |
| 지속   | 지속성 - 액세스 유지      |
| 회피   | 방어 회피 - 탐지 회피     |
| 디스크 | 발견 - 환경에 대해 학습   |
| EXFIL  | 유출 - 데이터 도용        |
| 영향   | 영향 - 손상 또는 중단     |

검토 중에 관리자가 ID를 할당합니다. 하나를 선택할 필요가 없습니다.

### 위험 수준

| 레벨     | 의미                                              |
| -------- | ------------------------------------------------- |
| **중요** | 전체 시스템 손상 또는 높은 가능성 + 심각한 영향   |
| **높음** | 심각한 피해 가능성 또는 중간 가능성 + 심각한 영향 |
| **중간** | 보통 위험 또는 낮은 가능성 + 높은 영향            |
| **낮음** | 가능성이 낮고 제한적인 영향                       |

위험 수준이 확실하지 않은 경우 영향을 설명하면 평가해 드리겠습니다.

## 검토 프로세스

1. **심사** - 48시간 이내에 새로운 제출물을 검토합니다.
2. **평가** - 타당성을 검증하고, ATLAS 매핑 및 위협 ID를 할당하고, 위험 수준을 검증합니다.
3. **문서화** - 모든 것이 형식화되고 완전하도록 보장합니다.
4. **병합** - 위협 모델 및 시각화에 추가됨
