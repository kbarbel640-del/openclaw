인프라를 코드로 사용하여 렌더링 시 OpenClaw를 배포합니다. 포함된 `render.yaml` Blueprint는 전체 스택, 서비스, 디스크, 환경 변수를 선언적으로 정의하므로 클릭 한 번으로 배포하고 코드와 함께 인프라 버전을 지정할 수 있습니다.

## 전제조건

- [렌더링 계정](https://render.com) (무료 등급 사용 가능)
- 선호하는 [모델 제공업체](/providers)의 API 키

## 렌더 블루프린트로 배포

[렌더링에 배포](https://render.com/deploy?repo=https://github.com/openclaw/openclaw)

이 링크를 클릭하면 다음이 수행됩니다.

1. 이 저장소의 루트에 있는 `render.yaml` 블루프린트에서 새 렌더 서비스를 생성합니다.
2. `SETUP_PASSWORD`를 설정하라는 메시지를 표시합니다.
3. Docker 이미지 빌드 및 배포

배포되면 서비스 URL은 `https://<service-name>.onrender.com` 패턴을 따릅니다.

## 청사진 이해하기

렌더 블루프린트는 인프라를 정의하는 YAML 파일입니다. 이 안에 있는 `render.yaml`
저장소는 OpenClaw를 실행하는 데 필요한 모든 것을 구성합니다.

```yaml
services:
  - type: web
    name: openclaw
    runtime: docker
    plan: starter
    healthCheckPath: /health
    envVars:
      - key: PORT
        value: "8080"
      - key: SETUP_PASSWORD
        sync: false # prompts during deploy
      - key: OPENCLAW_STATE_DIR
        value: /data/.openclaw
      - key: OPENCLAW_WORKSPACE_DIR
        value: /data/workspace
      - key: OPENCLAW_GATEWAY_TOKEN
        generateValue: true # auto-generates a secure token
    disk:
      name: openclaw-data
      mountPath: /data
      sizeGB: 1
```

사용된 주요 블루프린트 기능:

| 기능                  | 목적                                                              |
| --------------------- | ----------------------------------------------------------------- |
| `runtime: docker`     | 저장소의 Dockerfile에서 빌드                                      |
| `healthCheckPath`     | 렌더는 `/health`를 모니터링하고 비정상 인스턴스를 다시 시작합니다 |
| `sync: false`         | 배포 중 값에 대한 프롬프트 표시(비밀)                             |
| `generateValue: true` | 암호화된 보안 값을 자동 생성                                      |
| `disk`                | 재배포 후에도 지속되는 영구 스토리지                              |

## 요금제 선택

| 계획   | 스핀다운     | 디스크         |                   |
| ------ | ------------ | -------------- | ----------------- |
| 무료   | 15분 유휴 후 | 사용할 수 없음 | 테스트, 데모      |
| 스타터 | 절대         | 1GB 이상       | 개인용, 소규모 팀 |
| 표준+  | 절대         | 1GB 이상       | 제작, 다중 채널   |

블루프린트의 기본값은 `starter`입니다. 무료 등급을 사용하려면 포크에서 `plan: free`를 변경하세요.
`render.yaml` (그러나 참고: 영구 디스크가 없다는 것은 배포할 때마다 구성이 재설정된다는 의미입니다.)

## 배포 후

### 설정 마법사 완료

1. `https://<your-service>.onrender.com/setup`로 이동합니다.
2. `SETUP_PASSWORD`를 입력하세요.
3. 모델 제공자를 선택하고 API 키를 붙여넣으세요.
4. 선택적으로 메시징 채널 구성(Telegram, Discord, Slack)
5. **설정 실행**을 클릭합니다.

### 컨트롤 UI에 액세스

웹 대시보드는 `https://<your-service>.onrender.com/openclaw`에서 이용 가능합니다.

## 렌더 대시보드 기능

### 로그

**대시보드 → 서비스 → 로그**에서 실시간 로그를 확인하세요. 필터링 기준:

- 빌드 로그(Docker 이미지 생성)
- 로그 배포(서비스 시작)
- 런타임 로그(애플리케이션 출력)

### 셸 액세스

디버깅을 위해 **대시보드 → 서비스 → 셸**을 통해 셸 세션을 엽니다. 영구 디스크가 `/data`에 마운트되었습니다.

### 환경 변수

**대시보드 → 서비스 → 환경**에서 변수를 수정합니다. 변경 사항은 자동 재배포를 트리거합니다.

### 자동 배포

원래 OpenClaw 저장소를 사용하는 경우 Render는 OpenClaw를 자동 배포하지 않습니다. 업데이트하려면 대시보드에서 수동 블루프린트 동기화를 실행하세요.

## 맞춤 도메인

1. **대시보드 → 서비스 → 설정 → 사용자 정의 도메인**으로 이동합니다.
2. 도메인 추가
3. 지시에 따라 DNS를 구성합니다(CNAME을 `*.onrender.com`).
4. 렌더링은 TLS 인증서를 자동으로 프로비저닝합니다.

## 스케일링

렌더링은 수평 및 수직 크기 조정을 지원합니다.

- **수직**: 더 많은 CPU/RAM을 확보하기 위해 계획을 변경합니다.
- **수평**: 인스턴스 수 증가(표준 플랜 이상)

OpenClaw의 경우 일반적으로 수직 스케일링으로 충분합니다. 수평적 확장에는 고정 세션 또는 외부 상태 관리가 필요합니다.

## 백업 및 마이그레이션

언제든지 구성과 작업공간을 내보냅니다.

```
https://<your-service>.onrender.com/setup/export
```

그러면 모든 OpenClaw 호스트에서 복원할 수 있는 휴대용 백업이 다운로드됩니다.

## 문제 해결

### 서비스가 시작되지 않습니다

렌더 대시보드에서 배포 로그를 확인하세요. 일반적인 문제:

- `SETUP_PASSWORD` 누락 — 블루프린트에서 이를 묻는 메시지를 표시하지만 설정되어 있는지 확인하세요.
- 포트 불일치 - `PORT=8080`가 Dockerfile의 노출된 포트와 일치하는지 확인하세요.

### 느린 콜드 스타트(무료 등급)

15분 동안 활동이 없으면 무료 등급 서비스가 종료됩니다. 스핀다운 후 첫 번째 요청은 컨테이너가 시작되는 동안 몇 초 정도 걸립니다. 항상 켜져 있는 스타터 플랜으로 업그레이드하세요.

### 재배포 후 데이터 손실

이는 무료 등급(영구 디스크 없음)에서 발생합니다. 유료 플랜으로 업그레이드하거나
`/setup/export`를 통해 정기적으로 구성을 내보냅니다.

### 상태 확인 실패

렌더는 30초 이내에 `/health`로부터 200 응답을 예상합니다. 빌드가 성공했지만 배포가 실패하면 서비스를 시작하는 데 너무 오랜 시간이 걸릴 수 있습니다. 확인:

- 오류에 대한 로그 작성
- 컨테이너가 `docker build && docker run`를 사용하여 로컬에서 실행되는지 여부
