원클릭 템플릿을 사용하여 철도에 OpenClaw를 배포하고 브라우저에서 설정을 완료하세요.
이것은 가장 쉬운 "서버에 터미널 없음" 경로입니다. Railway가 사용자를 위해 게이트웨이를 실행합니다.
`/setup` 웹 마법사를 통해 모든 것을 구성합니다.

## 빠른 체크리스트(신규 사용자)

1. **철도에 배포**(아래)를 클릭합니다.
2. `/data`에 마운트된 **볼륨**을 추가합니다.
3. 필요한 **변수**(최소 `SETUP_PASSWORD`)를 설정합니다.
4. 포트 `8080`에서 **HTTP 프록시**를 활성화합니다.
5. `https://<your-railway-domain>/setup`를 열고 마법사를 종료합니다.

## 원클릭 배포

<a href="https://railway.com/deploy/clawdbot-railway-template" target="_blank" rel="noreferrer">
  철도에 배포
</a>

배포 후 **철도 → 서비스 → 설정 → 도메인**에서 공개 URL을 찾으세요.

철도는 다음 중 하나를 수행합니다.

- 생성된 도메인(종종 `https://<something>.up.railway.app`)을 제공하거나
- 사용자 정의 도메인을 연결한 경우 사용자 정의 도메인을 사용하십시오.

그런 다음 다음을 엽니다.

- `https://<your-railway-domain>/setup` — 설정 마법사(비밀번호로 보호됨)
- `https://<your-railway-domain>/openclaw` — 제어 UI

## 당신이 얻는 것

- 호스팅된 OpenClaw Gateway + 제어 UI
- `/setup`의 웹 설정 마법사(터미널 명령 없음)
- 구성/자격 증명/작업 공간이 재배포 후에도 유지되도록 철도 볼륨(`/data`)을 통한 영구 저장
- 나중에 철도에서 마이그레이션하기 위해 `/setup/export`에서 백업 내보내기

## 필수 철도 설정

### 공용 네트워킹

서비스에 대해 **HTTP 프록시**를 활성화합니다.

- 포트 : `8080`

### 볼륨(필수)

다음 위치에 마운트된 볼륨을 연결합니다.

- `/data`

### 변수

서비스에서 다음 변수를 설정합니다.

- `SETUP_PASSWORD` (필수)
- `PORT=8080` (필수 — 공용 네트워킹의 포트와 일치해야 함)
- `OPENCLAW_STATE_DIR=/data/.openclaw` (권장)
- `OPENCLAW_WORKSPACE_DIR=/data/workspace` (권장)
- `OPENCLAW_GATEWAY_TOKEN` (권장, 관리자 비밀번호로 취급)

## 설정 흐름

1. `https://<your-railway-domain>/setup`를 방문하여 `SETUP_PASSWORD`를 입력하세요.
2. 모델/인증 제공업체를 선택하고 키를 붙여넣습니다.
3. (선택사항) 텔레그램/Discord/Slack 토큰을 추가하세요.
4. **설정 실행**을 클릭합니다.

Telegram DM이 페어링으로 설정된 경우 설정 마법사가 페어링 코드를 승인할 수 있습니다.

## 채팅 토큰 받기

### 텔레그램 봇 토큰

1. 텔레그램의 메시지 `@BotFather`
2. `/newbot` 실행
3. 토큰을 복사합니다(`123456789:AA...`와 유사).
4. `/setup`에 붙여넣으세요.

### Discord 봇 토큰

1. [https://discord.com/developers/applications](https://discord.com/developers/applications)로 이동합니다.
2. **새 애플리케이션** → 이름 선택
3. **봇** → **봇 추가**
4. Bot → Privileged Gateway Intents(필수)에서 **메시지 콘텐츠 의도** 활성화(필수)(그렇지 않으면 봇이 시작 시 충돌함)
5. **봇 토큰**을 복사하여 `/setup`에 붙여넣습니다.
6. 봇을 서버에 초대합니다(OAuth2 URL 생성기, 범위: `bot`, `applications.commands`)

## 백업 및 마이그레이션

다음 위치에서 백업을 다운로드하세요.

- `https://<your-railway-domain>/setup/export`

그러면 OpenClaw 상태와 작업 공간이 내보내지므로 구성이나 메모리 손실 없이 다른 호스트로 마이그레이션할 수 있습니다.
