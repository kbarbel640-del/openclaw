# `amigo approvals`

Manage exec approvals for the **local host**, **gateway host**, or a **node host**.
By default, commands target the local approvals file on disk. Use `--gateway` to target the gateway, or `--node` to target a specific node.

Related:

- Exec approvals: [Exec approvals](/tools/exec-approvals)
- Nodes: [Nodes](/nodes)

## Common commands

```bash
amigo approvals get
amigo approvals get --node <id|name|ip>
amigo approvals get --gateway
```

## Replace approvals from a file

```bash
amigo approvals set --file ./exec-approvals.json
amigo approvals set --node <id|name|ip> --file ./exec-approvals.json
amigo approvals set --gateway --file ./exec-approvals.json
```

## Allowlist helpers

```bash
amigo approvals allowlist add "~/Projects/**/bin/rg"
amigo approvals allowlist add --agent main --node <id|name|ip> "/usr/bin/uptime"
amigo approvals allowlist add --agent "*" "/usr/bin/uname"

amigo approvals allowlist remove "~/Projects/**/bin/rg"
```

## Notes

- `--node` uses the same resolver as `amigo nodes` (id, name, ip, or id prefix).
- `--agent` defaults to `"*"`, which applies to all agents.
- The node host must advertise `system.execApprovals.get/set` (macOS app or headless node host).
- Approvals files are stored per host at `~/.amigo/exec-approvals.json`.
