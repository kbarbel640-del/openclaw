# Docker（選用）

Docker 是**選用**的。只有在你想要使用容器化的 Gateway 閘道器，或驗證 Docker 流程時才使用。

## Docker 適合我嗎？

- **是**：你想要一個隔離、可隨時丟棄的 Gateway 閘道器環境，或在沒有本機安裝的主機上執行 OpenClaw。
- **否**：你在自己的電腦上執行，只想要最快的開發迴圈。請改用一般的安裝流程。
- **沙箱隔離注意事項**：代理程式的沙箱隔離也會使用 Docker，但**不需要**整個 Gateway 閘道器都在 Docker 中執行。請參閱 [Sandboxing](/gateway/sandboxing)。

本指南涵蓋：

- 容器化 Gateway（在 Docker 中完整執行 OpenClaw）
- 每個工作階段的代理程式沙箱（主機上的 Gateway + Docker 隔離的代理程式工具）

沙箱隔離詳情：[Sandboxing](/gateway/sandboxing)

## 需求

- Docker Desktop（或 Docker Engine）+ Docker Compose v2
- 足夠的磁碟空間以存放映像檔與日誌

## 容器化 Gateway（Docker Compose）

### 快速開始（建議）

從儲存庫根目錄：

```bash
./docker-setup.sh
```

此腳本會：

- 建置 Gateway 映像檔
- 執行入門引導精靈
- 列印選用的提供者設定提示
- 透過 Docker Compose 啟動 Gateway
- 產生 Gateway 權杖並寫入 `.env`

選用的環境變數：

- `OPENCLAW_DOCKER_APT_PACKAGES` — 在建置期間安裝額外的 apt 套件
- `OPENCLAW_EXTRA_MOUNTS` — 新增額外的主機綁定掛載
- `OPENCLAW_HOME_VOLUME` — 以具名磁碟區保存 `/home/node`

完成後：

- 在瀏覽器中開啟 `http://127.0.0.1:18789/`。
- 將權杖貼到 Control UI（Settings → token）。
- 需要再次取得 URL？執行 `docker compose run --rm openclaw-cli dashboard --no-open`。

它會在主機上寫入設定與工作區：

- `~/.openclaw/`
- `~/.openclaw/workspace`

在 VPS 上執行？請參閱 [Hetzner（Docker VPS）](/install/hetzner)。

### 手動流程（compose）

```bash
docker build -t openclaw:local -f Dockerfile .
docker compose run --rm openclaw-cli onboard
docker compose up -d openclaw-gateway
```

注意：請從儲存庫根目錄執行 `docker compose ...`。如果你啟用了
`OPENCLAW_EXTRA_MOUNTS` 或 `OPENCLAW_HOME_VOLUME`，設定腳本會寫入
`docker-compose.extra.yml`；在其他地方執行 Compose 時請一併包含它：

```bash
docker compose -f docker-compose.yml -f docker-compose.extra.yml <command>
```

### Control UI 權杖 + 配對（Docker）

如果你看到「unauthorized」或「disconnected (1008): pairing required」，請取得
新的儀表板連結並核准瀏覽器裝置：

```bash
docker compose run --rm openclaw-cli dashboard --no-open
docker compose run --rm openclaw-cli devices list
docker compose run --rm openclaw-cli devices approve <requestId>
```

更多細節：[Dashboard](/web/dashboard)、[Devices](/cli/devices)。

### 額外掛載（選用）

如果你想將額外的主機目錄掛載到容器中，請在執行
`docker-setup.sh` 前設定 `OPENCLAW_EXTRA_MOUNTS`。此設定接受以逗號分隔的
Docker 綁定掛載清單，並會套用到 `openclaw-gateway` 與
`openclaw-cli`，方式是產生 `docker-compose.extra.yml`。

範例：

```bash
export OPENCLAW_EXTRA_MOUNTS="$HOME/.codex:/home/node/.codex:ro,$HOME/github:/home/node/github:rw"
./docker-setup.sh
```

注意事項：

- 在 macOS/Windows 上，路徑必須已與 Docker Desktop 共享。
- 如果你編輯了 `OPENCLAW_EXTRA_MOUNTS`，請重新執行 `docker-setup.sh` 以重新產生
  額外的 compose 檔案。
- `docker-compose.extra.yml` 是自動產生的，請勿手動編輯。

### 保存整個容器的 home（選用）

如果你希望 `/home/node` 在重新建立容器後仍能保留，請透過
`OPENCLAW_HOME_VOLUME` 設定具名磁碟區。這會建立一個 Docker 磁碟區並掛載到
`/home/node`，同時保留標準的設定/工作區綁定掛載。此處請使用
具名磁碟區（而非綁定路徑）；若要使用綁定掛載，請用
`OPENCLAW_EXTRA_MOUNTS`。

範例：

```bash
export OPENCLAW_HOME_VOLUME="openclaw_home"
./docker-setup.sh
```

你可以與額外掛載一起使用：

```bash
export OPENCLAW_HOME_VOLUME="openclaw_home"
export OPENCLAW_EXTRA_MOUNTS="$HOME/.codex:/home/node/.codex:ro,$HOME/github:/home/node/github:rw"
./docker-setup.sh
```

注意事項：

- 如果你變更了 `OPENCLAW_HOME_VOLUME`，請重新執行 `docker-setup.sh` 以重新產生
  額外的 compose 檔案。
- 具名磁碟區會一直保留，直到使用 `docker volume rm <name>` 移除為止。

### 安裝額外的 apt 套件（選用）

如果你需要在映像檔內安裝系統套件（例如建置工具或媒體函式庫），請在執行
`docker-setup.sh` 前設定 `OPENCLAW_DOCKER_APT_PACKAGES`。
這會在映像檔建置期間安裝套件，因此即使刪除容器也會保留。

範例：

```bash
export OPENCLAW_DOCKER_APT_PACKAGES="ffmpeg build-essential"
./docker-setup.sh
```

注意事項：

- 此設定接受以空白分隔的 apt 套件名稱清單。
- 如果你變更了 `OPENCLAW_DOCKER_APT_PACKAGES`，請重新執行 `docker-setup.sh` 以重新建置
  映像檔。

### 進階使用者／全功能容器（選擇性）

預設的 Docker 映像檔以**安全優先**為設計，並以非 root 的 `node`
使用者執行。這能縮小攻擊面，但也代表：

- 執行期間無法安裝系統套件
- 預設不含 Homebrew
- 不包含 Chromium/Playwright 瀏覽器

如果你想要功能更完整的容器，請使用以下選擇性設定：

1. **保存 `/home/node`**，讓瀏覽器下載與工具快取得以保留：

```bash
export OPENCLAW_HOME_VOLUME="openclaw_home"
./docker-setup.sh
```

2. **將系統相依套件烘焙進映像檔**（可重複、可持久）：

```bash
export OPENCLAW_DOCKER_APT_PACKAGES="git curl jq"
./docker-setup.sh
```

3. **在不使用 `npx` 的情況下安裝 Playwright 瀏覽器**（避免 npm 覆寫衝突）：

```bash
docker compose run --rm openclaw-cli \
  node /app/node_modules/playwright-core/cli.js install chromium
```

如果你需要 Playwright 安裝系統相依套件，請使用
`OPENCLAW_DOCKER_APT_PACKAGES` 重新建置映像檔，而不是在執行期間使用 `--with-deps`。

4. **保存 Playwright 瀏覽器下載內容**：

- 在 `docker-compose.yml` 中設定 `PLAYWRIGHT_BROWSERS_PATH=/home/node/.cache/ms-playwright`。
- 透過 `OPENCLAW_HOME_VOLUME` 確保 `/home/node` 會被保留，或透過
  `OPENCLAW_EXTRA_MOUNTS` 掛載 `/home/node/.cache/ms-playwright`。

### 權限 + EACCES

映像檔以 `node`（uid 1000）執行。若你在
`/home/node/.openclaw` 上看到權限錯誤，請確保你的主機綁定掛載由 uid 1000 擁有。

範例（Linux 主機）：

```bash
sudo chown -R 1000:1000 /path/to/openclaw-config /path/to/openclaw-workspace
```

若你選擇為了方便而以 root 執行，則需接受安全性上的取捨。

### 更快的重新建置（建議）

為了加快重新建置速度，請將 Dockerfile 依序排列，使相依層能被快取。
這能避免在鎖定檔未變更時重跑 `pnpm install`：

```dockerfile
FROM node:22-bookworm

# Install Bun (required for build scripts)
RUN curl -fsSL https://bun.sh/install | bash
ENV PATH="/root/.bun/bin:${PATH}"

RUN corepack enable

WORKDIR /app

# Cache dependencies unless package metadata changes
COPY package.json pnpm-lock.yaml pnpm-workspace.yaml .npmrc ./
COPY ui/package.json ./ui/package.json
COPY scripts ./scripts

RUN pnpm install --frozen-lockfile

COPY . .
RUN pnpm build
RUN pnpm ui:install
RUN pnpm ui:build

ENV NODE_ENV=production

CMD ["node","dist/index.js"]
```

### 注意事項

- Gateway 綁定預設為 `lan`，供容器使用。
- Dockerfile 的 CMD 使用 `--allow-unconfigured`；若掛載的設定使用 `gateway.mode` 而非 `local`，仍可啟動。若要強制防護，請覆寫 CMD。
- Gateway 容器是工作階段的事實來源（`~/.openclaw/agents/<agentId>/sessions/`）。

## 代理程式沙箱（主機 Gateway + Docker 工具）

深入說明：[Sandboxing](/gateway/sandboxing)

### 功能說明

啟用 `agents.defaults.sandbox` 時，**非主要工作階段**會在 Docker
容器內執行工具。Gateway 仍留在你的主機上，但工具執行會被隔離：

- 範圍：預設為 `"agent"`（每個代理程式一個容器 + 工作區）
- 範圍：`"session"` 可達到每個工作階段的隔離
- 每個範圍的工作區資料夾掛載於 `/workspace`
- 選用的代理程式工作區存取（`agents.defaults.sandbox.workspaceAccess`）
- 工具允許/拒絕政策（拒絕優先）
- 傳入的媒體會被複製到目前的沙箱工作區（`media/inbound/*`），以便工具讀取（搭配 `workspaceAccess: "rw"` 時，會落在代理程式工作區）

警告：`scope: "shared"` 會停用跨工作階段隔離。所有工作階段將共用
同一個容器與同一個工作區。

### 每個代理程式的沙箱設定檔（多代理）

若你使用多代理路由，每個代理程式都可以覆寫沙箱與工具設定：
`agents.list[].sandbox` 與 `agents.list[].tools`（以及 `agents.list[].tools.sandbox.tools`）。這讓你能在
同一個 Gateway 中混合不同的存取層級：

- 完整存取（個人代理）
- 唯讀工具 + 唯讀工作區（家庭/工作代理）
- 無檔案系統/殼層工具（公開代理）

請參閱 [Multi-Agent Sandbox & Tools](/multi-agent-sandbox-tools) 以取得範例、
優先順序與疑難排解。

### 預設行為

- 映像檔：`openclaw-sandbox:bookworm-slim`
- 每個代理程式一個容器
- 代理程式工作區存取：`workspaceAccess: "none"`（預設）使用 `~/.openclaw/sandboxes`
  - `"ro"` 會將沙箱工作區保留在 `/workspace`，並以唯讀方式掛載代理程式工作區至 `/agent`（停用 `write`/`edit`/`apply_patch`）
  - `"rw"` 會以讀寫方式掛載代理程式工作區至 `/workspace`
- 自動清理：閒置 > 24 小時 或 存在時間 > 7 天
- 網路：預設為 `none`（若需要對外連線請明確選擇）
- 預設允許：`exec`、`process`、`read`、`write`、`edit`、`sessions_list`、`sessions_history`、`sessions_send`、`sessions_spawn`、`session_status`
- 預設拒絕：`browser`、`canvas`、`nodes`、`cron`、`discord`、`gateway`

### 啟用沙箱隔離

若你打算在 `setupCommand` 中安裝套件，請注意：

- 預設 `docker.network` 為 `"none"`（無對外連線）。
- `readOnlyRoot: true` 會阻擋套件安裝。
- `user` 必須為 root 才能進行 `apt-get`（省略 `user` 或設定 `user: "0:0"`）。
  當 `setupCommand`（或 docker 設定）變更時，OpenClaw 會自動重新建立容器，
  除非容器**最近被使用**（約 5 分鐘內）。熱容器會記錄警告並顯示確切的
  `openclaw sandbox recreate ...` 指令。

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main", // off | non-main | all
        scope: "agent", // session | agent | shared (agent is default)
        workspaceAccess: "none", // none | ro | rw
        workspaceRoot: "~/.openclaw/sandboxes",
        docker: {
          image: "openclaw-sandbox:bookworm-slim",
          workdir: "/workspace",
          readOnlyRoot: true,
          tmpfs: ["/tmp", "/var/tmp", "/run"],
          network: "none",
          user: "1000:1000",
          capDrop: ["ALL"],
          env: { LANG: "C.UTF-8" },
          setupCommand: "apt-get update && apt-get install -y git curl jq",
          pidsLimit: 256,
          memory: "1g",
          memorySwap: "2g",
          cpus: 1,
          ulimits: {
            nofile: { soft: 1024, hard: 2048 },
            nproc: 256,
          },
          seccompProfile: "/path/to/seccomp.json",
          apparmorProfile: "openclaw-sandbox",
          dns: ["1.1.1.1", "8.8.8.8"],
          extraHosts: ["internal.service:10.0.0.5"],
        },
        prune: {
          idleHours: 24, // 0 disables idle pruning
          maxAgeDays: 7, // 0 disables max-age pruning
        },
      },
    },
  },
  tools: {
    sandbox: {
      tools: {
        allow: [
          "exec",
          "process",
          "read",
          "write",
          "edit",
          "sessions_list",
          "sessions_history",
          "sessions_send",
          "sessions_spawn",
          "session_status",
        ],
        deny: ["browser", "canvas", "nodes", "cron", "discord", "gateway"],
      },
    },
  },
}
```

強化設定位於 `agents.defaults.sandbox.docker` 之下：
`network`、`user`、`pidsLimit`、`memory`、`memorySwap`、`cpus`、`ulimits`、
`seccompProfile`、`apparmorProfile`、`dns`、`extraHosts`。

多代理：可透過 `agents.list[].sandbox.{docker,browser,prune}.*` 為每個代理程式覆寫 `agents.defaults.sandbox.{docker,browser,prune}.*`
（當 `agents.defaults.sandbox.scope` / `agents.list[].sandbox.scope` 為 `"shared"` 時會被忽略）。

### 沙箱通用映像檔（選用）

如果你想要包含常見建置工具（Node、Go、Rust 等）的沙箱映像檔，請建置通用映像檔：

```bash

```

```json5
{
  agents: {
    defaults: {
      sandbox: { docker: { image: "openclaw-sandbox-common:bookworm-slim" } },
    },
  },

```

### 沙箱瀏覽器映像檔

若要在沙箱中執行瀏覽器工具，請建置瀏覽器映像檔：

```bash
scripts/sandbox-browser-setup.sh
```

這會使用
`Dockerfile.sandbox-browser` 建置 `openclaw-sandbox-browser:bookworm-slim`。容器會以啟用 CDP 的方式執行 Chromium，並提供選用的 noVNC 觀察器（透過 Xvfb 顯示的有頭模式）。

注意事項：

- 有頭模式（Xvfb）相較於無頭模式，較能降低被機器人阻擋的機率。
- 仍可透過設定 `agents.defaults.sandbox.browser.headless=true` 使用無頭模式。
- 不需要完整的桌面環境（GNOME）；Xvfb 會提供顯示。

使用設定：

```json5
{
  agents: {
    defaults: {
      sandbox: {
        browser: { enabled: true },
      },
    },
  },
}
```

自訂瀏覽器映像檔：

```json5
{
  agents: {
    defaults: {
      sandbox: { browser: { image: "my-openclaw-browser" } },
    },
  },
}
```

啟用後，代理程式會收到：

- 沙箱瀏覽器控制 URL（供 `browser` 工具使用）
- noVNC URL（若已啟用且 headless=false）

請記住：若你使用工具允許清單，請加入 `browser`（並從拒絕清單移除），否則工具仍會被封鎖。
清理規則（`agents.defaults.sandbox.prune`）也會套用到瀏覽器容器。

### 工具政策（允許/拒絕）

- `deny` 的優先順序高於 `allow`。
- 若 `allow` 為空：所有工具（除了拒絕清單）皆可使用。
- 若 `allow` 非空：僅 `allow` 中的工具可用（扣除拒絕清單）。

### 清理策略

兩個調整項目：

- `prune.idleHours`：移除 X 小時內未使用的容器（0 = 停用）
- `prune.maxAgeDays`：移除存在超過 X 天的容器（0 = 停用）

範例：

- 保留忙碌的工作階段，但限制生命週期：
  `idleHours: 24`、`maxAgeDays: 7`
- 永不清理：
  `idleHours: 0`、`maxAgeDays: 0`

### 安全性注意事項

- 嚴格隔離僅適用於**工具**（exec/read/write/edit/apply_patch）。
- 僅限主機的工具（如 browser/camera/canvas）預設為封鎖。
- 在沙箱中允許 `browser` 會**破壞隔離**（瀏覽器會在主機上執行）。

## 疑難排解

- 映像檔缺失：請使用 [`scripts/sandbox-setup.sh`](https://github.com/openclaw/openclaw/blob/main/scripts/sandbox-setup.sh) 建置，或設定 `agents.defaults.sandbox.docker.image`。
- 容器未執行：會在每個工作階段需要時自動建立。
- 沙箱中的權限錯誤：請將 `docker.user` 設為與你掛載之工作區擁有權相符的 UID:GID（或對工作區資料夾執行 chown）。
- 找不到自訂工具：OpenClaw 以 `sh -lc`（登入殼層）執行指令，會載入
  `/etc/profile` 並可能重設 PATH。請設定 `docker.env.PATH` 以預先加入你的
  自訂工具路徑（例如 `/custom/bin:/usr/local/share/npm-global/bin`），或在 Dockerfile 中於
  `/etc/profile.d/` 底下加入腳本。
