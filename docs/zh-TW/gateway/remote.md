# 遠端存取（SSH、通道與 tailnet）

此儲存庫支援「透過 SSH 的遠端存取」，做法是在專用主機（桌機／伺服器）上維持單一 Gateway（主節點）運行，並讓用戶端連線至該 Gateway。

- 對 **操作人員（你／macOS 應用程式）**：SSH 通道是通用的備援方案。
- 對 **節點（iOS/Android 與未來裝置）**：連線到 Gateway **WebSocket**（LAN／tailnet，或視需要走 SSH 通道）。

## 核心概念

- Gateway WebSocket 會綁定在你設定的連接埠之 **loopback**（預設為 18789）。
- 遠端使用時，透過 SSH 轉發該 loopback 連接埠（或使用 tailnet／VPN 以減少通道需求）。

## 常見的 VPN／tailnet 架構（代理程式所在位置）

把 **Gateway 主機** 視為「代理程式所在處」。它擁有工作階段、驗證設定檔、頻道與狀態。
你的筆電／桌機（以及節點）會連線到該主機。

### 1) tailnet 中的常駐 Gateway（VPS 或家用伺服器）

在持續運作的主機上執行 Gateway，並透過 **Tailscale** 或 SSH 存取。

- **最佳體驗：** 保持 `gateway.bind: "loopback"`，並使用 **Tailscale Serve** 提供控制 UI。
- **備援方案：** 維持 loopback + 從任何需要存取的機器建立 SSH 通道。
- **範例：** [exe.dev](/install/exe-dev)（簡易 VM）或 [Hetzner](/install/hetzner)（正式環境 VPS）。

當你的筆電經常休眠、但你希望代理程式長時間運作時，這是理想選擇。

### 2) 家用桌機執行 Gateway，筆電作為遠端控制

筆電 **不** 執行代理程式，而是進行遠端連線：

- 使用 macOS 應用程式的 **Remote over SSH** 模式（設定 → 一般 →「OpenClaw runs」）。
- 應用程式會開啟並管理通道，因此 WebChat 與健康檢查可「即開即用」。

操作手冊：[macOS 遠端存取](/platforms/mac/remote)。

### 3) 筆電執行 Gateway，其他機器遠端存取

讓 Gateway 保持在本機，但安全地對外提供：

- 從其他機器對筆電建立 SSH 通道，或
- 使用 Tailscale Serve 提供控制 UI，並讓 Gateway 僅綁定 loopback。

指南：[Tailscale](/gateway/tailscale) 與 [Web 概覽](/web)。

## 指令流程（哪些元件在哪裡執行）

單一 gateway 服務擁有狀態與頻道；節點是周邊設備。

流程範例（Telegram → 節點）：

- Telegram 訊息抵達 **Gateway**。
- Gateway 執行 **agent**，並決定是否呼叫節點工具。
- Gateway 透過 Gateway WebSocket（`node.*` RPC）呼叫 **節點**。
- 節點回傳結果；Gateway 再回覆到 Telegram。

備註：

- **節點不會執行 gateway 服務。** 除非你刻意執行隔離的設定檔（請見 [Multiple gateways](/gateway/multiple-gateways)），否則每台主機只應有一個 gateway。
- macOS 應用程式的「node mode」只是透過 Gateway WebSocket 的節點用戶端。

## SSH 通道（CLI + 工具）

建立通往遠端 Gateway WS 的本機通道：

```bash
ssh -N -L 18789:127.0.0.1:18789 user@host
```

通道建立後：

- `openclaw health` 與 `openclaw status --deep` 會透過 `ws://127.0.0.1:18789` 存取遠端 gateway。
- 需要時，`openclaw gateway {status,health,send,agent,call}` 也可透過 `--url` 指向轉發後的 URL。

注意：將 `18789` 替換為你設定的 `gateway.port`（或 `--port`/`OPENCLAW_GATEWAY_PORT`）。
注意：當你傳入 `--url` 時，CLI 不會回退使用設定檔或環境變數中的憑證。
請明確包含 `--token` 或 `--password`。未提供明確憑證將視為錯誤。

## CLI 遠端預設值

你可以將遠端目標設為預設，讓 CLI 指令自動使用：

```json5
{
  gateway: {
    mode: "remote",
    remote: {
      url: "ws://127.0.0.1:18789",
      token: "your-token",
    },
  },
}
```

當 gateway 僅綁定 loopback 時，請將 URL 維持在 `ws://127.0.0.1:18789`，並先開啟 SSH 通道。

## 透過 SSH 的聊天 UI

WebChat 不再使用獨立的 HTTP 連接埠。SwiftUI 聊天 UI 會直接連線至 Gateway WebSocket。

- 透過 SSH 轉發 `18789`（見上文），然後讓用戶端連線至 `ws://127.0.0.1:18789`。
- 在 macOS 上，建議使用應用程式的「Remote over SSH」模式，該模式會自動管理通道。

## macOS 應用程式「Remote over SSH」

macOS 選單列應用程式可端到端完成相同設定（遠端狀態檢查、WebChat 與語音喚醒轉發）。

操作手冊：[macOS 遠端存取](/platforms/mac/remote)。

## 安全性規則（遠端／VPN）

簡要原則：**除非確定需要對外綁定，否則請讓 Gateway 僅綁定 loopback。**

- **Loopback + SSH／Tailscale Serve** 是最安全的預設（無公開曝露）。
- **非 loopback 綁定**（`lan`/`tailnet`/`custom`，或在無法使用 loopback 時的 `auto`）必須使用驗證權杖／密碼。
- `gateway.remote.token` **僅** 用於遠端 CLI 呼叫 — 它 **不會** 啟用本機驗證。
- 使用 `wss://` 時，`gateway.remote.tlsFingerprint` 會釘選遠端 TLS 憑證。
- **Tailscale Serve** 可在 `gateway.auth.allowTailscale: true` 時透過身分標頭進行驗證。
  若你希望改用權杖／密碼，請將其設為 `false`。
- 將瀏覽器控制視同操作人員存取：僅限 tailnet + 審慎的節點配對。

深入說明：[Security](/gateway/security)。
