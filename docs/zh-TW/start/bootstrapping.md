# 代理程式啟動

啟動是 **首次執行** 的儀式，用於準備代理程式的工作區並蒐集身分詳細資料。它發生在入門引導之後，當代理程式第一次啟動時進行。

## 啟動會做什麼

在代理程式第一次執行時，OpenClaw 會為工作區（預設
`~/.openclaw/workspace`）進行啟動：

- 播種 `AGENTS.md`、`BOOTSTRAP.md`、`IDENTITY.md`、`USER.md`。
- 執行一個簡短的問答儀式（一次一個問題）。
- 將身分 + 偏好寫入 `IDENTITY.md`、`USER.md`、`SOUL.md`。
- 完成後移除 `BOOTSTRAP.md`，以確保只執行一次。

## 在哪裡執行

啟動一律在 **Gateway 閘道器主機** 上執行。若 macOS 應用程式連線到遠端 Gateway 閘道器，工作區與啟動檔案會位於該遠端機器上。

<Note>
當 Gateway 閘道器在另一台機器上執行時，請在 gateway 主機上編輯工作區檔案（例如 `user@gateway-host:~/.openclaw/workspace`）。
</Note>
