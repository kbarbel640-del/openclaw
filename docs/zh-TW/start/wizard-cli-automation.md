# CLI 自動化

使用 `--non-interactive` 來自動化 `openclaw onboard`。

<Note>
`--json` 不代表非互動模式。請在腳本中使用 `--non-interactive`（以及 `--workspace`）。
</Note>

## 基準非互動式範例

```bash
openclaw onboard --non-interactive \
  --mode local \
  --auth-choice apiKey \
  --anthropic-api-key "$ANTHROPIC_API_KEY" \
  --gateway-port 18789 \
  --gateway-bind loopback \
  --install-daemon \
  --daemon-runtime node \
  --skip-skills
```

加入 `--json` 以取得機器可讀的摘要。

## 新增另一個代理程式

使用 `openclaw agents add <name>` 建立一個具有獨立工作區、工作階段與驗證設定檔的代理程式。未搭配 `--workspace` 執行時，會啟動精靈。

```bash
openclaw agents add work \
  --workspace ~/.openclaw/workspace-work \
  --model openai/gpt-5.2 \
  --bind whatsapp:biz \
  --non-interactive \
  --json
```

它會設定的項目：

- `agents.list[].name`
- `agents.list[].workspace`
- `agents.list[].agentDir`

注意事項：

- 預設工作區遵循 `~/.openclaw/workspace-<agentId>`。
- 加入 `bindings` 以路由傳入訊息（精靈也可以完成此設定）。
- 非互動式旗標：`--model`、`--agent-dir`、`--bind`、`--non-interactive`。
