# `openclaw system`

Gateway 閘道器的系統層級輔助工具：排入系統事件、控制心跳，
以及檢視存在狀態。

## Common commands

```bash
openclaw system event --text "Check for urgent follow-ups" --mode now
openclaw system heartbeat enable
openclaw system heartbeat last
openclaw system presence
```

## `system event`

在 **main** 工作階段上排入一個系統事件。下一次心跳會將其注入為提示中的一行 `System:`。使用 `--mode now` 以立即觸發心跳；
`next-heartbeat` 會等待下一個排程的節點。

旗標：

- `--text <text>`：必要的系統事件文字。
- `--mode <mode>`：`now` 或 `next-heartbeat`（預設）。
- `--json`：機器可讀的輸出。

## `system heartbeat last|enable|disable`

心跳控制：

- `last`：顯示最後一次心跳事件。
- `enable`：重新開啟心跳（若先前已停用，請使用此項）。
- `disable`：暫停心跳。

旗標：

- `--json`：機器可讀的輸出。

## `system presence`

列出 Gateway 閘道器目前所知的系統存在狀態項目（節點、實例，以及類似的狀態行）。

旗標：

- `--json`：機器可讀的輸出。

## Notes

- 需要一個可依你目前的設定（本機或遠端）連線的運作中 Gateway 閘道器。
- 系統事件為暫時性，重新啟動後不會保留。
