# モデル設定（検討）

このドキュメントは、将来のモデル設定に関する **アイデア** をまとめたものです。これは出荷される仕様ではありません。現在の動作については、以下を参照してください。

- [モデル](/concepts/models)
- [モデルのフェイルオーバー](/concepts/model-failover)
- [OAuth + プロファイル](/concepts/oauth)

## 動機

運用者が求めるもの:

- プロバイダーごとに複数の認証プロファイル（個人用 vs 仕事用）。
- 予測可能なフォールバックを備えた、シンプルな `/model` の選択。
- テキストモデルと画像対応モデルの明確な分離。

## 可能な方向性（ハイレベル）

- モデル選択をシンプルに保つ: `provider/model` に任意のエイリアスを追加。
- プロバイダーに、明示的な順序を持つ複数の認証プロファイルを持たせる。
- すべてのセッションが一貫してフェイルオーバーするよう、グローバルなフォールバックリストを使用する。
- 明示的に設定された場合にのみ、画像ルーティングを上書きする。
