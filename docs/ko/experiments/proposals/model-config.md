# 모델 설정 (탐색)

이 문서는 향후 모델 구성을 위한 **아이디어**를 정리합니다. 이는
출시된 사양이 아닙니다. 현재 동작은 다음을 참고합니다:

- [모델](/concepts/models)
- [모델 페일오버](/concepts/model-failover)
- [OAuth + 프로필](/concepts/oauth)

## 동기

운영자는 다음을 원합니다:

- 프로바이더별 여러 인증 프로필(개인용 vs 업무용).
- 예측 가능한 폴백을 갖춘 간단한 `/model` 선택.
- 텍스트 모델과 이미지 지원 모델 간의 명확한 분리.

## 가능한 방향(개요)

- 모델 선택은 단순하게 유지: 선택적으로 별칭을 포함한 `provider/model`.
- 프로바이더가 명시적 순서를 가진 여러 인증 프로필을 갖도록 합니다.
- 모든 세션이 일관되게 페일오버하도록 전역 폴백 목록을 사용합니다.
- 명시적으로 구성된 경우에만 이미지 라우팅을 오버라이드합니다.

## 미해결 질문

- 프로필 로테이션은 프로바이더별이어야 할까요, 모델별이어야 할까요?
- UI 는 세션에 대한 프로필 선택을 어떻게 노출해야 할까요?
- 레거시 설정 키에서 가장 안전한 마이그레이션 경로는 무엇인가요?
