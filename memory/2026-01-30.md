### Gateway Auth Flow
OpenClaw uses challenge-response auth:
1. Client connects
2. Gateway sends `connect.challenge` with nonce
3. Client sends `connect` frame (NOT `hello`) with nonce + auth token
4. Gateway responds with `hello_ok`
