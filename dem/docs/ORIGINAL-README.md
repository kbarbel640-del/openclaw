## Core Principles

### AI Controls
- **Multi-Agent Validation**: Every agent action validated by peer agents (CEO/COO/CCO model)
- **JIT Permissions**: Agents request time-bound, scoped access to tools and data
- **Sandboxed Execution**: Reversible operations execute in isolation before commit
- **Human-in-the-Loop**: Critical decisions escalate to DevSOC human operators
- **On-Chain Audit Trail**: All agent decisions logged immutably via DAO governance

### Data Controls
- **Zero Trust Data Access**: Agents prove need-to-know for every data request
- **PII Classification**: Automatic detection and handling of sensitive data
- **Consent Verification**: CCO agent validates compliance before data operations
- **Encryption at Rest/Transit**: All secrets and sensitive outputs encrypted
- **Data Retention Policies**: Automatic expiration of investigation artifacts

## Pipeline Integration

### GitHub Actions

```yaml
name: DevSOC Security Pipeline

on:
  pull_request:
    branches: [main]
  push:
    branches: [main]

jobs:
  threat-model:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Run Threat Modeling Agent
        uses: diabolus-ex-machina/threat-modeler@v1
        with:
          iac_paths: |
            ./terraform
            ./kubernetes
            ./docker
          output_format: sarif
          create_pr: true
          devsoc_review: true

      - name: Upload SARIF
        uses: github/codeql-action/upload-sarif@v3
        with:
          sarif_file: threat-model.sarif

      - name: Queue for DevSOC Review
        if: steps.threat-model.outputs.critical_threats > 0
        uses: diabolus-ex-machina/devsoc-queue@v1
        with:
          severity: critical
          assignee: security-team
```

## Getting Started

### Prerequisites

```bash
# Core dependencies
docker --version  # >= 24.0
python --version  # >= 3.11
node --version    # >= 20.0

# Optional: Local LLM
ollama --version  # For self-hosted models
```

### Quick Start

```bash
# Clone the repository
git clone https://github.com/your-org/diabolus-ex-machina.git
cd diabolus-ex-machina

# Start the platform
docker compose up -d

# Run threat model on your infrastructure
dem threat-model analyze ./path/to/your/iac

# View results in dashboard
open http://localhost:8080/devsoc
```

### Configuration

```yaml
# dem.yaml - DevSOC Configuration
devsoc:
  # AI Controls
  ai:
    model: deepseek-r1:7b  # or openai:gpt-4, anthropic:claude-3
    validation:
      require_multi_agent: true
      min_confidence: 0.85
      human_review_threshold: critical

  # Data Controls
  data:
    pii_detection: true
    encryption: aes-256-gcm
    retention_days: 90
    audit_log: true

  # Pipeline Integration
  pipeline:
    github:
      enabled: true
      auto_pr: true
      required_review: true
    gitlab:
      enabled: false
    azure_devops:
      enabled: false

  # Agent Configuration
  agents:
    threat_modeler:
      enabled: true
      frameworks: [stride, pasta]
      iac_types: [terraform, kubernetes, docker]
    secret_scanner:
      enabled: true
      entropy_threshold: 4.5
```
