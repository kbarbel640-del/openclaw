# PREFLIGHT.md — Чеклист перед КАЖДЫМ ответом

⚠️ Прочитай это ПЕРВЫМ. Не отвечай пока не пройдёшь чеклист.

## Обязательно:
1. ☐ Проверь скиллы — есть ли подходящий для задачи? (creative-thought-partner для brainstorm, coding-agent для кода, и т.д.)
2. ☐ Код = Claude Code. Любые изменения в проектах — только через него.
3. ☐ Стрим прогресса. Влад в Телеграме. Каждый этап — короткое сообщение.
4. ☐ Если работа с проектом — прочитай memory/learnings/{project}.md

## По типу задачи:
| Задача | Действие |
|--------|----------|
| Brainstorm/идея | → скилл creative-thought-partner + читать/обновлять `notes/areas/brainstorm-*.md` |
| Код/изменения | → скилл `the-loop` → следуй протоколу THE LOOP |
| Баг/проблема | → сначала логи, потом гипотезы |
| Проект | → прочитать learnings проекта |
| Вопрос о прошлом | → memory_search |

## После ответа:
1. ☐ Записать в memory/YYYY-MM-DD.md
2. ☐ Если была ошибка — записать в memory/learnings/global.md
3. ☐ Если ошибка в проекте — записать в memory/learnings/{project}.md

## Reverse prompting (периодически):
- "Что полезного я могу сделать на основе того что знаю о тебе?"
- Предлагать идеи которые Влад не просил
- Замечать паттерны → предлагать автоматизацию
- НЕ делать ничего внешнего без одобрения (draft, не send)

## Self-healing:
- При проблеме — попробовать минимум 5 подходов перед тем как просить помощь
- Проверить логи, docs, web search, GitHub issues
- Документировать решение в learnings
